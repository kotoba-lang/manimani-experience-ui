# Sources and interpretation

このrepoは公式資料に書かれた事実と、manimaniへの設計上の推論を区別します。

## Official sources

- [任天堂：UI/UXデザイン職](https://www.nintendo.co.jp/jobs/introduction/design/work03.html) — 幅広い利用者へ文字情報に頼りすぎず伝えること、「ツール」ではなく「娯楽」として魅力も設計すること。
- [任天堂：「ワクワク」や「うれしい」をデザイン](https://www.nintendo.co.jp/jobs/keyword/150.html) — 好奇心や期待を自然に引き出し、実用性と魅力を両立するUI/UXの事例。
- [Nintendo Labo developer interview, part 1](https://www.nintendo.com/fr-fr/News/2018/Avril/Entrevue-avec-les-developpeurs-de-Nintendo-Labo-1re-partie-le-concept-1363026.html) — 無地では組立時の表裏や完成像が伝わらず、利用者テストを通して印刷、工程、形状を改善した事例。
- [Nintendo Labo developer interview, part 2](https://www.nintendo.com/au/news-and-articles/nintendo-labo-developer-interview-part-2-prototyping-and-development/) — prototypeを重ね、「作る・修理する・変更する・仕組みを発見する」ことを体験へ含めた事例。
- [Apple Human Interface Guidelines](https://developer.apple.com/design/human-interface-guidelines/) — platform conventions、interaction、accessibilityの基礎参照。
- [W3C Web Content Accessibility Guidelines 2.2](https://www.w3.org/TR/WCAG22/) — Web実装のアクセシビリティ基準。

## Our interpretation

次は任天堂が公表した規定ではなく、上記からmanimani向けに導いた独自原則です。

- Foundation UIとExperience UIの二層モデル
- Kinetic Companion UI / 随伴玩具UIという名称と配分
- まにまに君の物理motion、報酬文法、agent stateとの対応
- Issue → Agent → PR → Actions → Receiptのjourney
- system dynamics loopをWalkする螺旋topology
